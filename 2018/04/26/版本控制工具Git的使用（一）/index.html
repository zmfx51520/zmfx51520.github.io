版本控制工具Git的使用（一）

发表于 2018-04-26

Git和SVN的对比

1）Git是分布式的，SVN是集中式；
2）Git每个历史版本存储完整的文件，SVN存储文件的差异；
3）Git可离线完成大部分操作，SVN则必须与中央服务器进行网络交互；
4）Git有着优雅的分支和合并功能；
5）Git有着更强的撤销修改和修改版本历史的能力；
6）Git速度更快，效率更好；

Git的三个分区

工作区

我们在站点目录下的修改添加和删除等操作都是在工作区内完成的

暂存区

当我们把工作区内的文件git add的时候，就是把内容由工作区添加到暂存区中了

历史版本区

暂存区中的内容，经过git commit以后，就提交到了历史仓库区

配置相关的命令

添加属性

添加user.name和user.email
git config –global user.name zmfx
git config –global user.email zmfx51520@outlook.com

添加一个新的email
git config –global –add user.email 242781365@qq.com

查看属性

获得配置中name属性的值， –get 可省略
git config –get user.name
git config user.name

得到全局配置中的所有属性及属性值

删除属性

删除指定的属性名对应的属性值
git config –global –unset user.email 242781365@qq.com

当属性名对应的值唯一时，可以省略属性值，删除整个属性

版本回退

这里说一下回退的机制，是通过一个HEAD指针实现的
回退只是将指针指向了之前的版本，而存储的内容并没有改变，
所以回退之后，还是可以反悔的，只需要再将指针指向更新的版本即可（当然要知道版本号）

向前回退N个版本，N代表^符号的个数
git reset –hard HEAD^ 的意思就是回退一个版本

切换到指定版本号的版本
版本号可以通过上面说到的 git log 命令获得
git reset –hard HEAD^ 的意思就是回退一个版本
